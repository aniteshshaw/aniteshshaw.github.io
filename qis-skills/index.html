Visibility Control

You control what's public and when

Not every skill needs to be public on day one. The catalog reads from a single config file — change visibility, schedule a launch date, and the site updates automatically.

data/skills-config.json

"featured"

Highlighted in the catalog with a gold badge. Your flagship skills — always visible, always installable.

"public"

Visible and installable. Standard release — shows with a green badge, links to GitHub.

"preview"

Visible with a blue badge but marked as early access. Use for skills you're testing with early adopters.

"coming-soon"

Visible with a yellow badge, no install link. Teases upcoming skills to build anticipation.

"hidden"

Completely removed from the public catalog. For skills still in development or internal-only.

"publishAfter"

Optional ISO date (e.g. "2026-09-01"). Auto-promotes preview/coming-soon skills to public on that date.

// Example: hide a skill until launch { "id": "qis-ios-automation", "visibility": "coming-soon", "publishAfter": "2026-09-01" } // Example: feature-flag a utility { "id": "qis-diagram-forge", "visibility": "hidden" }
